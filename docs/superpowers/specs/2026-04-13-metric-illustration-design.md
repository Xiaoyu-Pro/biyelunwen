# 指标波动示意图设计说明

## 1. 目标

为毕业论文补充一张“指标数据波动与组成形式”示意图。图像风格参考用户提供的四联图，重点表达原始指标序列可拆分为趋势项、周期项和残差项，而不是呈现真实实验数据。

## 2. 设计结论

采用“手工合成示意数据后直接绘图”的方式实现，不依赖真实指标文件，也不依赖复杂时序分解库。脚本内部直接生成四部分数据：

- `data`：总体指标序列，表现为基线、缓慢趋势、周期波动和噪声的叠加
- `trend`：平滑趋势项，用于展示指标整体变化方向
- `seasonal`：周期项，用于展示重复性的波动模式
- `remainder`：残差项，用于展示随机扰动和少量异常点

这种方式的优点是：

- 可控性高，能稳定生成与参考图接近的论文插图
- 依赖简单，优先使用 `numpy` 与 `matplotlib`
- 适合作为论文示意图，避免读者误解为真实实验结果

## 3. 输出物

实现阶段将新增一个可直接运行的 Python 脚本，并输出两种图片格式：

- `PNG`：便于预览和直接插入文档
- `SVG`：便于论文排版时保持矢量清晰度

预期文件组织如下：

- `scripts/plot_metric_illustration.py`
- `figures/metric_illustration.png`
- `figures/metric_illustration.svg`

## 4. 绘图要求

图像采用四行单列布局，分别展示：

1. 原始指标序列 `data`
2. 趋势项 `trend`
3. 周期项 `seasonal`
4. 残差项 `remainder`

风格要求如下：

- 整体风格贴近参考图，适合论文展示
- 以黑灰色线条和浅色背景为主，避免花哨配色
- 左侧显示分面标签，便于表达四类组成部分
- 横轴展示时间步或采样点，纵轴展示指标值
- 保留适度网格线，增强阅读性

## 5. 非目标

本次实现不包含以下内容：

- 不接入真实 CSV/Excel 指标数据
- 不引入论文业务语义标签，如具体服务名或实验编号
- 不尝试严格复刻参考图中的每个细节，仅保留核心表达形式

## 6. 成功标准

满足以下条件则认为实现完成：

- 运行一次脚本即可生成完整四联图
- 图片能直观看出“趋势 + 周期 + 噪声”的组成关系
- 生成结果适合直接放入毕业论文作为示意配图
- 脚本结构简洁，后续若要替换标题、标签或随机种子可方便修改
