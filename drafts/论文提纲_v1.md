# 面向多源异构时序数据的系统异常检测技术研究

## 建议目录

### 第一章 绪论

#### 1.1 研究背景与意义

- 微服务系统复杂度提升，异常检测难度增加。
- 指标、日志、调用链分别从性能、语义、结构角度描述系统状态。
- 单一模态异常检测存在信息利用不足的问题。

#### 1.2 国内外研究现状

- 基于指标数据的时间序列异常检测。
- 基于日志数据的异常检测。
- 基于调用链/图结构的异常检测。
- 多源异构数据融合异常检测。
- 现有研究的不足总结。

#### 1.3 研究问题与挑战

- 多源异构数据结构差异大，统一建模困难。
- 单模态表示能力有限，难以充分覆盖异常模式。
- 跨模态融合容易引入噪声或掩盖关键特征。

#### 1.4 本文研究内容与贡献

- 提出统一的多源异构异常检测框架。
- 设计面向 Metric / Log / Trace 的差异化特征增强与特征提取方法。
- 提出基于门控机制的异构特征融合方法。
- 在公开数据集上进行对比、消融和泛化分析。

#### 1.5 论文组织结构
